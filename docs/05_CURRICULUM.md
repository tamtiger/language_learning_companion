# Chương trình học

> Học theo năng lực. Luyện theo nhiệm vụ. Áp dụng vào công việc thực tế.

---

# Mục tiêu

Lesson không phải là nơi học ngữ pháp hay ghi nhớ từ vựng.

Mỗi Lesson được thiết kế để giúp bạn hoàn thành **một nhiệm vụ giao tiếp cụ thể**.

Sau mỗi Lesson, bạn phải trả lời được:

- Tôi làm được việc gì mới?
- Tôi có thể áp dụng việc này ở đâu?
- Tôi cần luyện thêm điều gì?

Nếu không trả lời được, Lesson chưa hoàn thành.

# Lesson

Mỗi Lesson chỉ giải quyết **một mục tiêu duy nhất**.

Ví dụ:

Lesson

```
Giải thích API
```

Không dạy:

- Docker
- Kubernetes
- Database

Chỉ tập trung:

**Giải thích API bằng tiếng Anh.**

---

# Cấu trúc một Lesson

Mỗi Lesson đều sử dụng cùng một cấu trúc.

## 1. Mục tiêu

Sau bài học, bạn sẽ làm được điều gì?

Ví dụ:

> Giải thích REST API trong 3 phút.

---

## 2. Kiến thức cần thiết

Bao gồm:

- Vocabulary
- Expressions
- Sentence Patterns
- Pronunciation

Chỉ học những gì phục vụ mục tiêu.

---

## 3. Input

Nguồn học.

Ví dụ:

- Video
- Podcast
- Article

---

## 4. Shadowing

Bắt chước cách diễn đạt.

Không cần học thuộc.

---

## 5. Speaking Task

Thực hiện nhiệm vụ.

Ví dụ:

> Explain your API.

Không nhìn tài liệu.

---

## 6. AI Feedback

Nhờ AI nhận xét:

- Pronunciation
- Grammar
- Vocabulary
- Fluency

---

## 7. Reflection

Trả lời:

- Mình làm tốt điều gì?
- Mình còn gặp khó khăn gì?

---

# Nguyên tắc thiết kế Lesson

## Một Lesson chỉ có một mục tiêu

Không dạy quá nhiều thứ.

Ví dụ:

Sai

```
Lesson

API

Database

Docker

Git
```

Đúng

```
Lesson

Explain REST API
```

---

## Học để sử dụng

Không học để ghi nhớ.

Ví dụ:

Sai

```
Học 30 từ mới.
```

Đúng

```
Giải thích API bằng 30 từ đã học.
```

---

## Luôn có Speaking

Nếu Lesson không có phần Speaking.

Đó chưa phải Lesson hoàn chỉnh.

---

## Luôn có Feedback

Không luyện một mình.

Luôn có:

- AI
- Giáo viên
- Bạn học

Để sửa lỗi.

# Quan hệ giữa Lesson và Mission

Lesson giúp bạn hoàn thành Mission.

Ví dụ:

Mission

```
Ngày đầu tiên đi làm
```

Bao gồm:

Lesson

- Giới thiệu bản thân
- Small Talk
- Giới thiệu dự án
- Daily Stand-up
- Hỏi Requirement

Hoàn thành tất cả Lesson.

↓

Hoàn thành Mission.

---

# Quan hệ giữa Lesson và Assessment

Assessment không kiểm tra bạn đã học Lesson chưa.

Assessment kiểm tra:

Bạn có hoàn thành được nhiệm vụ hay không.

Ví dụ:

Lesson

```
Explain API
```

Assessment

```
Giải thích API cho một đồng nghiệp mới.
```

---

# Tiêu chí hoàn thành Lesson

Một Lesson chỉ được xem là hoàn thành khi:

□ Hiểu nội dung.

□ Shadow được.

□ Hoàn thành Speaking Task.

□ Nhận Feedback.

□ Sửa các lỗi quan trọng.

□ Có thể thực hiện lại mà không cần nhìn tài liệu.

Nếu chưa đạt, tiếp tục luyện.

Không chuyển sang Lesson tiếp theo.

---

# Tiêu chí hoàn thành Module

Một Module được hoàn thành khi bạn có thể thực hiện thành công Mission tương ứng.

Ví dụ:

Module

Developer Communication

↓

Mission

Implement a New Feature

↓

Bạn có thể:

- Nhận yêu cầu.
- Hỏi lại Requirement.
- Giải thích giải pháp.
- Tạo Pull Request.
- Review Code.

Hoàn toàn bằng tiếng Anh.

# Định nghĩa một Lesson tốt

Một Lesson tốt không phải là Lesson có nhiều kiến thức.

Một Lesson tốt là Lesson giúp bạn làm được một việc mới bằng tiếng Anh.

Nếu sau bài học bạn có thể giao tiếp tốt hơn trong công việc, Lesson đã hoàn thành đúng mục tiêu.
