# Vì sao âm cuối quan trọng?

Tiếng Việt hầu như không phát âm đầy đủ phụ âm cuối như tiếng Anh.

Vì vậy, nhiều người có xu hướng nói:

```
I like it.

↓

I like i...
```

Hoặc:

```
worked

↓

work
```

Đối với người bản xứ, việc thiếu âm cuối có thể khiến:

- Khó hiểu.
- Hiểu nhầm thì của câu.
- Hiểu nhầm số ít và số nhiều.
- Hiểu nhầm từ.

---

# Những âm cuối cần ưu tiên

Không cần học tất cả cùng lúc.

Trong Lesson này, hãy tập trung vào:

```
/t/

/d/

/k/

/p/

/s/

/z/

/m/

/n/

/ŋ/
```

Đây là những âm xuất hiện rất thường xuyên.

---

# Âm /t/

Ví dụ:

```
cat

not

right

light

internet
```

Lưu ý:

Âm /t/ ở cuối thường rất nhẹ.

Không đọc thành:

```
cát

nót
```

Chỉ cần chạm đầu lưỡi và dừng.

---

# Âm /d/

Ví dụ:

```
good

need

read

called
```

Đừng biến /d/ thành /t/.

---

# Âm /k/

Ví dụ:

```
book

work

look

network
```

Đây là âm rất nhiều người Việt bỏ mất.

---

# Âm /p/

Ví dụ:

```
stop

help

app

develop
```

Môi khép lại nhưng không bật mạnh như tiếng Việt.

---

# Âm /s/ và /z/

Đây là hai âm rất quan trọng vì liên quan đến:

- Danh từ số nhiều.
- Chia động từ.
- Sở hữu.

Ví dụ:

```
book

books

developer

developers

work

works
```

Nếu bỏ /s/, người nghe có thể hiểu sai.

---

# Âm /m/ và /n/

Ví dụ:

```
team

problem

open

login

design
```

Luyện phát âm rõ phần cuối.

---

# Âm /ŋ/

Ví dụ:

```
working

learning

speaking

engineering
```

Không đọc:

```
work-in

learn-in
```

Hãy kéo nhẹ âm mũi ở cuối.

---

# Quy tắc phát âm đuôi -ed
 
 Đây là kiến thức quan trọng khi giao tiếp về các hành động trong quá khứ (như *fixed*, *deployed*, *tested*). Đuôi "-ed" có 3 cách phát âm:
 
 1. Phát âm là **/ɪd/** khi động từ kết thúc bằng âm **/t/** hoặc **/d/**.
    - Ví dụ: *test* ➔ *tested* (/ˈtestɪd/), *start* ➔ *started* (/ˈstɑːrtɪd/), *need* ➔ *needed* (/ˈniːdɪd/).
 2. Phát âm là **/t/** khi động từ kết thúc bằng các phụ âm vô thanh (như **/p/, /k/, /f/, /θ/, /s/, /ʃ/, /tʃ/**).
    - Ví dụ: *fix* ➔ *fixed* (/fɪkst/), *work* ➔ *worked* (/wɜːrkt/), *develop* ➔ *developed* (/dɪˈveləpt/).
 3. Phát âm là **/d/** khi động từ kết thúc bằng các âm hữu thanh còn lại (như **/b/, /g/, /v/, /z/, /ð/, /dʒ/, /m/, /n/, /l/, /r/** và các nguyên âm).
    - Ví dụ: *deploy* ➔ *deployed* (/dɪˈplɔɪd/), *call* ➔ *called* (/kɔːld/), *solve* ➔ *solved* (/sɒlvd/).

 # Minimal Pairs
 
 Đọc và nghe sự khác biệt.
 
 | Phụ âm vô thanh (Voiceless) | Phụ âm hữu thanh (Voiced) |
 |------------|------------------|
 | back (/bæk/) | bag (/bæɡ/) |
 | cap (/kæp/) | cab (/kæb/) |
 | bet (/bet/) | bed (/bed/) |
 | rice (/raɪs/) | rise (/raɪz/) |
 | sink (/sɪŋk/) | sing (/sɪŋ/) |

Mục tiêu không phải đọc nhanh.

Mà là đọc đúng.

# Speaking Practice

Đọc chậm các câu sau:

```
I work at a software company.

We need to fix this bug.

The project has three services.

Please check the last commit.

This feature was deployed yesterday.
```

Đọc 3 lần.

Lần 1:

Chậm.

Lần 2:

Tự nhiên.

Lần 3:

Ghi âm.

# Những lỗi thường gặp

Người Việt thường:

- Bỏ hoàn toàn âm cuối.
- Đọc âm cuối quá mạnh.
- Không phân biệt /s/ và /z/.
- Không phát âm "ed".
- Không phát âm "ing" đúng.

Nếu bạn gặp các lỗi trên, hãy ghi vào:

# Thử thách cuối bài

Ghi âm và đọc đoạn sau:

> Last week, our backend team fixed several critical bugs. We tested the new feature, deployed it to production, and monitored the system carefully. Everything worked as expected.
